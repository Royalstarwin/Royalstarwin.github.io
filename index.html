钉钉注册公司及获取配置文件

发表于 2018-06-07

登录钉钉官网

https://www.dingtalk.com/

注册公司

  1. 点击注册公司，跳转公司注册页面
  2. 填写手机号码、图片验证码、短信验证码
  3. 点击下一步
  4. 填写：
    企业(团队)名称
    行业类型
    人员规模
    管理密码
    联系人姓名
    勾选已阅读《钉钉服务协议》
  5. 点击注册

创建部门

  1. 点击通讯录
  2. 点击添加子部门，输入部门名称，点确定

创建微应用

  1. 点击工作台 -> 自建应用
  2. 输入：
    应用logo
    应用名称
    功能介绍
    首页地址 (首页地址写应用对应外网地址，可联系运维，格式参照截图)
    可见范围 (选刚在通讯录录入的部门)
  3. 点击提交

点开创建成功的微应用，可以得到AgentID参数

管理微应用

  1. 点击应用开发 (在自建应用附近)
  2. 点击生成CorpSecret
  3. 输入名称、关联的部门 (选刚在通讯录录入的部门)
  4. 点击确认

这里可以得到CorpId和CorpSecret参数

  1. 点击左侧自助工具
  2. 点击创建扫码登录应用授权
  3. 输入名称、描述、授权LOGO地址 (静态图片地址)、回调域名
  4. 点击确定

这里可以得到appId和appSecret参数

配置工号：

选择通讯录→组织架构→刚才添加的子部门→选择添加成员→设置成员姓名，部门，手机号，工号（工号为中心操作员的grbh）

钉钉使用方式

从手机上打开钉钉，点击底部菜单栏中间的【工作】，顶部下拉框选择之前创建的公司，下划屏幕，找【其他应用】，点击之前创建的微应用，点击【待我审批】

钉钉技术方案

发表于 2018-05-09

1.整体方案

​ 钉钉开发选择使用当下流行的React+nodejs的开发模式；使用ReactRedux作为数据层管理，完善组件间通信；使用webpack作为模块打包工具，精简静态资源；使用中心板的各个接口作为数据源，实现手机钉钉和中心板的交互；根据阿里的官方文档，发布上线到钉钉微应用中。

​ 与传统的jQuery+Ajax开发方式不同，以React作为前端框架，Nodejs作为中转后台，是以前端发请求给后端，后端写路由的方式进行前后端分离开发的。

​ React+nodejs的优势：React构建的是一个路由控制的单页面应用，入口文件只有一个index.html，避免了页面跳转所带来的问题；使用react可以减少dom操作，通过操作虚拟dom（Virtual DOM）来更新视图，避免页面的reflow，repaint带来的性能问题；前端代码组件化，方便重用，用jsx语法糖将js，html甚至css写在一起形成组件；nodejs作为中转后台，使前后端分离，前端执行页面逻辑、视图，nodejs作为后端请求数据；nodejs为钉钉程序过滤掉无效数据，将有效数据传递给前台，优化http请求。

2.前端代码

2.1 前端框架选择

​ 选择当下最流行的三大框架之一，由facebook开源的React。与传统web应用的不同，React为了尽可能减少对DOM的操作，使用操作虚拟dom（用来描述dom的javascript对象）的方式，在通过diff算法转化成真实dom，提升页面性能。使用jsx语法糖的写法，也就是js内定义的一套XML语法，可以解
析出目标js代码,颠覆传统js写法。更加语义化，允许自定义标签及组件。更加直观，标签处理方式，更加可读。关注分离,模块化，性能优秀。

2.2 前端路由React-Router

使用React-Router构建了前端的单页面应用，分离前后端，前端负责界面显示，后端负责数据存储和计算，各司其职；减轻服务器压力，服务器只用出数据就可以，不用管展示逻辑和页面合成，提升用户体验；同时还实现了与多页面应用类似的功能：

​ 1）在地址栏显示的URL总能够反映出视图展示的东西。
​ 2）能够成功地使用浏览器的返回和前进按钮。
​ 3）能够直接的使用相关联的URL导航到某个特别的视图。

​ 使用history第三方库，管理js运行时的会话历史记录。history抽象出各种环境中的差异，管理历史堆栈，导航，确认导航以及在会话之间保持状态。

2.3 React-Redux管理数据层

Redux是针对JavaScript应用的可预测状态容器

​ 因为react的state是所有组件共享的状态，state的改变也会影响到所有用到相关state的组件导致页面刷新；多个react组件间通信困难。

​ 所以使用React-Redux来解决这些问题。它将所有组件需要用到的state，放在唯一的一个store中，每个组件都可以订阅（subscribe）某个或者某些state，当这些state发生变化时store会通知所有订阅相关state的组件重新渲染。而改变state的方法从杂乱无章的setState()，变成了事件（action）派发（dispatch）的方式。当某些方法创建action后，将他传入store，store将oldState和action交给相关的reducer处理，处理后用newState将其替换。提高了程序的稳定性的同时，也提高了代码的可读性，将智能组件和UI组件分工明确，智能组件处理逻辑，UI组件负责内容呈现。

2.4 数据请求与异步处理方案

​ 使用成熟的基于promise的http库—Axios，来处理ajax数据请求；异步处理方案选择号称“异步编程终级解决方案”的 async/await，解决了传统方案中回调嵌套回调的回调地狱问题，提高性能，代码可读性更高；解决了Promise的then()链中多个请求结果互相依赖传递参数困难的问题，更好的语义性。

2.5 前端代码的构建与打包

​ 使用官方推荐的脚手架工具create-react-app，构建前端基础构架。使用webpack作为前端打包工具，将代码分割，使用babel和cssloader等编译ES6,ES7写法和css提高兼容性，最后静态文件精简，减小服务端加载静态资源的压力。

3.后端代码

3.1 为什么nodejs做中转后台？

1）React是高性能的前端框架，不擅长数据请求和处理数据，所以交给nodejs来做。
2）路由设计，控制逻辑，数据请求和处理交给node后端，UI呈现交给React。
3）nodejs擅长处理I/O，善于处理数据相互调用的情况。（用A接口的数据调用B接口，继续向下调用）
4）修复接口数据，请求后端数据不尽理想，node中转后台整理数据结构，过滤无效数据为前端提供更搞笑的数据。

3.2 后端框架Koa2

使用Express原班人马打造，更小巧的Koa2。相比与“稳健”的Express，Koa2完美支持了ES6，ES7的新特性，自动转码，减少了配置文件和对编译工具的依赖。
koa-route 采用的是restful设计模式，路由的模块化 路由规则是域名+模块+方法，例如：http://localhost:3000/approval/todo 它的优势在于结构清晰、符合标准、易于理解、扩展方便。
middleware中间件，koa2对网络请求采用了中间件的形式处理,中间件可以介入请求和相应的处理,是一个轻量级的模块,每个中间件负责完成某个特定的功能。通过next函数联系，更加方便的处理请求。

3.3 Socket.io实现消息推送

Socket.io封装了html5中的WebSocket新特性和轮询等方法，socket通讯可以近似的认为是一个长连接。不同的xhr或者http请求是一次性的，请求完成之后通道关闭，而Socket.io可以将通道保存，通道的两端可以相互喊话，大量使用在通讯功能之中。

3.4 pm2管理node程序进程

pm2是node.js生产过程管理器，它保证app一直运行，任何一个线程挂掉将自动重启；监测所有已启动的流程，方便程序管理，提高node性能，简化集群操作，实施拓展集群。

4.总结

钉钉是以React为前端框架，nodejs为中转后台，数据源依赖中心板的app。设计模式是前后端分离的，前端组件是模块化、复用性强的；js语法使用ES6/ES7+babel编译的形式，让js代码更精简；前端自动化打包工具使用webpack，高效的压缩静态资源；最后前端文件和后端文件整合，pm2管理发布上线。

人聪明有想法，工作努力又充满热情，工作外喜欢自己开发一些APP，从百度离职后去了一家创业公司。再次相遇是2017年去新西兰旅游的时候，他说在新西兰已经自驾了20天了，这次旅行没有计划，开到哪儿是哪儿，不想呆了再回国。

我问怎么请的假，他说：辞职了。后来才得知，他经历的大型创业公司上市后套现了一笔丰厚的资金，在云南买了个小院，和老婆离开了北京。

现在，他的朋友圈里全是无人机的航拍，以及全球自驾游的风景。

朋友B

校招加入百度，4年以火箭般的速度升到了当时设计最高级别；曾在Ixdc连着3年当讲师及主持人，出书、博客大V、馒头商学院讲师等等。她有很多头衔，是个典型的斜杠青年，也是一个永远精力充沛的女强人。

当达到百度事业至高点时，她选择了离职。当时我们不理解，问她为什么在顶峰期不继续留在这儿？

她回答：Always stay out of your comfort zone，翻译过来就是：经常离开舒适区，以挖掘自己最大的价值。

最近见她的时候，身怀六甲脚蹬高跟鞋，优雅地进行创业宣讲。气宇轩昂，激情无限，一点没有孕傻的状态。

这两个朋友的经历与下面我们要谈的互联网人的职业规划有关。

换工作的时机

选择大于努力，每一次人生的选择（包括择校、就业、婚姻）都会决定和改变你未来的人生方向。

经济学中有一个名词叫做『机会成本』，是指企业为从事某项经营活动而放弃另一项经营活动的机会。就业同理，当你选择了一项工作后，同时也就放弃了其他多项工作的选择权。

基于此，换工作需慎重考虑。

马爸爸说过一句话：

“离职要么是钱受委屈，要么是人受委屈。”

但我认为这只是一部分，随着个体独立思考能力的不断完善，换工作时还需考虑更多维度：

1. 当你对手头工作缺乏热情的时候

对工作的激情是一个人在职场的内在驱动力。

就像上面讲述的朋友A和B，无论什么时候见到总能被他们的状态所感染。有没有发现团队中总有影响力和感染力强的人，跟他们在一起共事，自己会潜移默化变得更积极。

相反地，当一个人没有激情的时候，往往会怀念过去…

怀念刚进公司那会为了一个细节，晚上加班到12点还跟打了鸡血一样！怀念上线后在各种反馈平台看到用户关于新功能的评论，开心的跳脚~

曾经的你在乎每一个用户的每一句赞美和吐槽，在乎每一个功能埋点的数据波动；甚至把从0做起来的产品当做自己的孩子对待：当有其他角色提出产品价值>用户体验价值的功能时，会极力辩驳、寻找更舒适的着陆方式。

那个时候，你就像一团烈火看得清自己的前方，也照亮了周围的同事。但随着经验、时间、时机等导致这团火熄灭的时候，整个世界，就暗下来了。

所以，当热情消逝的时候，别再浪费自己的时间和生命了，可以换一个有兴趣的领域，重新点燃内心的火焰吧！

2. 当你深处舒适区的时候

很多人在工作几年后，面对手头工作完全可以通过直觉和经验轻而易举的达到目标；每天的工作变得重复且没有挑战，这个时候就要注意，你已进入舒适区。

这里普及3种状态，分别是：舒适区、学习区和焦虑区。

  1. 舒适区：是熟练掌握技能，沉浸在自我享受的状态中；
  2. 学习区：是最佳状态，在本职领域知识充沛的情况下，涉猎少量未知领域，通过学习补足短板，不断拓展能力圈的状态；
  3. 焦虑区：是面对未知领域知识陷入焦虑、恐慌、担忧，同时无法进步的状态。

这三种状态中学习区是最佳状态，过于焦虑或过于悠闲对职业未来发展都有影响。

人性天生喜欢懒惰、舒适，所以逃离舒适区是反人性的，但也只有逃离舒适区才能让人有所成长。很多时候人们在某条路上走的太久了，习惯成自然，于是忘记了最开始为什么要选择这条路。勿忘初心，方得始终。

3. 当你到达事业顶峰的时候

很多人不明白事业顶峰期为什么要离职？

  1. 高落点才能有更高的起点。当你在这个舞台发挥了极致能力并获得成就后，你的职业状态是最佳的，凭借这种最佳的状态，可以成为下一个职业道路上的高起点。
  2. 职业的第二曲线原理。混沌大学李善友讲创业公司的核心是寻找“第二曲线”，职业生涯同理，在巅峰时刻跳出第一曲线的依赖，找到第二曲线才是倍速成长的重要原因。

现实中，相反很多人离职是在工作低谷，这个时候我建议需要分析低谷的原因，如果这个原因在下一个公司还可能复现（如人际关系等），那必须正面面对去解决，一定要逼自己过了这个坎儿。

另一方面，低谷时候换工作，因前一段工作的因果关系还未做了结，自身会带着一些阴郁的因素，必然会影响下一份工作。

4. 当你的工作无法实现自我价值的时候

产品大师梁宁老师说过：

一个人的内核就是对存在感的定义，很多人需要在不同领域实现自我价值，寻求存在感。

工作中自我价值的体现方式有很多种，如：能力提升、项目成功、被领导认可、升职加薪等。但当这份工作没有成长空间、无法实现自我价值的时候就需要考虑如何度过瓶颈期了。

除了以上4个内在原因外，可能还有诸多外在原因，包括：健康、家庭、薪酬福利、工作氛围、公司规模、离家距离、部门变动、公司倒闭、合同到期等都可能成为换工作的时机。

但请一定要将眼光放长远，不要为了短期的利益做决定，考虑换工作的原因一定要剖析清楚，每一步踩扎实了才能走的更远。

如何选择下一份工作？

很多人都以”钱多活少”为标准，但我认为活少未必是个正确的抉择依据，活少会让人进入舒适区。对我而言，选择有以下几个维度：

1. 感兴趣/有热情的行业

兴趣是起点、热情是推动剂，只有完美融合才能形成稳定的内在驱动力。譬如：曾经的百度同事辉哥是一个产品控，喜欢高品质的产品并愿意为之付费，喜欢研究驾驶工具，从奔驰到特斯拉，之后他也毅然决然进入了车和家，这个意欲打造未来城市智能交通工具的企业，对电动车和变革城市交通的热情使他做出了这个选择。

2. 大势所趋的行业

小米CEO雷军说过：站在风口上猪都会飞起来。

几年前BAT借着互联网打拼出自己天地，如今头条、快手等又借助短视频开辟新领域，小米的未来智能家庭布局也占据了AI时代的先手。此外，很多传统企业也借助互联网+实现企业转型。

因此，在挑选行业时，只有懂势、借势才能把握未来。就像我的A朋友，踏上了风口浪尖的行业，很快实现了财务自由。

3. 牛人团队

《穷查理宝典》中提到睿智老头芒格对工作的3点感悟：

  1. 别兜售你自己不会购买的东西；
  2. 别为你不尊敬和钦佩的人工作；
  3. 只跟你喜欢的人共事。

查理·芒格说自己很幸运，他和巴菲特共事了几十年。

同样的，当和你同一个团队中有牛人时，从ta身上学到的会比自己摸索更快。借助一句话，钱可以慢慢挣，但好导师绝对不能错过。

4. 发展空间

足够大的舞台才能让人的潜能发挥的淋漓尽致，所以一个公司的发展空间是决策时的必要条件。进入公司后还能走多远多高都需要在选择时考虑清楚，因为天花板的高度决定了你可能在下一个公司奋斗的时长。之后是薪资股票、涨薪周期、Title、离家近、福利待遇、年假情况等。

设置 侧栏

默认情况下，侧栏仅在文章页面（拥有目录列表）时才显示，并放置于右侧位置。 可以通过修改 主题配置文件 中的 sidebar 字段来控制侧栏的行为。侧栏的设置包括两个部分，其一是侧栏的位置， 其二是侧栏显示的时机。

  1. 设置侧栏的位置，修改 sidebar.position 的值，支持的选项有：

    • left - 靠左放置
    • right - 靠右放置

    目前仅 Pisces Scheme 支持 position 配置。影响版本5.0.0及更低版本。

  2. 设置侧栏显示的时机，修改 sidebar.display 的值，支持的选项有：

    • post - 默认行为，在文章页面（拥有目录列表）时显示
    • always - 在所有页面中都显示
    • hide - 在所有页面中都隐藏（可以手动展开）
    • remove - 完全移除

    已知侧栏在 use motion: false 的情况下不会展示。 影响版本5.0.0及更低版本。
